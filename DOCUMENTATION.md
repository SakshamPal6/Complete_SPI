# Detailed Documentation: Full-Duplex SPI Master/Slave

## 1. Introduction

This project implements a configurable 4-wire SPI system in SystemVerilog. It contains a master, a slave, a top-level wrapper that connects them, and a self-checking verification environment.

SPI, or Serial Peripheral Interface, is a synchronous serial communication protocol commonly used between a controller device and peripheral devices such as ADCs, DACs, sensors, memories, shift registers, switches, and multiplexers. In a standard 4-wire SPI connection, the bus contains:

- `SCLK`: Serial clock generated by the master
- `CS`: Chip select, usually active low
- `MOSI`: Master out, slave in
- `MISO`: Master in, slave out

SPI is full-duplex, which means both sides can transmit and receive simultaneously. While the master shifts data out on `MOSI`, the slave shifts data out on `MISO`. The clock edge used for sampling and shifting is controlled by the `CPOL` and `CPHA` mode settings.

The theory summary in this document is based on standard SPI behavior and the Analog Devices article [Introduction to SPI Interface](https://www.analog.com/en/resources/analog-dialogue/articles/introduction-to-spi-interface.html).

## 2. Supported Features

The design supports:

- Full-duplex master/slave SPI transfer
- Configurable data width using the `DW` parameter
- Default 12-bit data words
- All four CPOL/CPHA SPI modes
- Active-low chip select
- Master-generated serial clock
- Configurable SCLK divider
- MSB-first transmission
- One-cycle master `done` pulse
- Slave `done` pulse when a complete word has been received
- Self-checking testbench across modes 0, 1, 2, and 3

## 3. SPI Mode Theory

The two important SPI mode parameters are:

- `CPOL`: Clock polarity. It defines the idle level of `SCLK`.
- `CPHA`: Clock phase. It defines whether data is sampled on the leading or trailing clock edge.

The four standard SPI modes are:

| SPI Mode | CPOL | CPHA | SCLK Idle Level | Sample Edge | Shift Edge |
|---|---:|---:|---|---|---|
| 0 | 0 | 0 | Low | Leading/rising | Trailing/falling |
| 1 | 0 | 1 | Low | Trailing/falling | Leading/rising |
| 2 | 1 | 0 | High | Leading/falling | Trailing/rising |
| 3 | 1 | 1 | High | Trailing/rising | Leading/falling |

In this RTL:

- A **leading edge** is the transition away from the idle clock level.
- A **trailing edge** is the transition back to the idle clock level.
- For `CPHA = 0`, data is sampled on the leading edge and shifted on the trailing edge.
- For `CPHA = 1`, data is shifted on the leading edge and sampled on the trailing edge.

## 4. Source File: `design.sv`

`design.sv` contains the complete RTL:

- `spi_master`
- `spi_slave`
- `top`

The `top` module connects the master and slave together for a simple loop-style SPI system where the master and slave exchange parallel input words.

## 5. Master Module

### Module Name

```systemverilog
spi_master
```

### Parameters

| Parameter | Default | Description |
|---|---:|---|
| `DW` | 12 | Data width in bits |
| `CPOL` | 0 | Clock idle polarity |
| `CPHA` | 0 | Clock phase |
| `CLK_DIV` | 4 | Number of system-clock cycles per SCLK half-period |

### Ports

| Port | Direction | Description |
|---|---|---|
| `clk` | Input | System clock |
| `rst` | Input | Synchronous reset |
| `newd` | Input | One-cycle start pulse for a new transfer |
| `din` | Input | Parallel word transmitted from master to slave |
| `miso` | Input | Serial data from slave |
| `sclk` | Output | SPI serial clock |
| `cs` | Output | Active-low chip select |
| `mosi` | Output | Serial data to slave |
| `dout` | Output | Parallel word received from slave |
| `done` | Output | One-cycle transfer-complete pulse |

### Master FSM

The master uses four states:

| State | Purpose |
|---|---|
| `IDLE` | Waits for `newd`, keeps `cs` high and SCLK disabled |
| `LOAD` | Latches `din`, pulls `cs` low, and prepares the first bit if needed |
| `XFER` | Performs serial shifting and sampling |
| `FIN` | Stops SCLK, releases `cs`, updates `dout`, and pulses `done` |

### SCLK Generation

The master contains a clock divider. When `sclk_en` is low, `sclk` is forced to the configured idle value:

```systemverilog
sclk <= CPOL[0];
```

When `sclk_en` is high, the divider counts up to `CLK_DIV - 1`, toggles `sclk`, and generates one-cycle internal edge flags:

- `leading_edge`
- `trailing_edge`

These flags let the transfer FSM handle SPI modes without duplicating all edge logic.

### Master Transfer Behavior

For `CPHA = 0`:

- The first `MOSI` bit is placed on the bus before the first sampling edge.
- The master samples `MISO` on each leading edge.
- The master updates `MOSI` on each trailing edge.

For `CPHA = 1`:

- The master places data on `MOSI` on each leading edge.
- The master samples `MISO` on each trailing edge.

Data is transmitted MSB first using `tx_shift[DW-1]` first, then the next lower bits.

## 6. Slave Module

### Module Name

```systemverilog
spi_slave
```

### Parameters

| Parameter | Default | Description |
|---|---:|---|
| `DW` | 12 | Data width in bits |
| `CPOL` | 0 | Clock idle polarity |
| `CPHA` | 0 | Clock phase |

### Ports

| Port | Direction | Description |
|---|---|---|
| `sclk` | Input | SPI clock from master |
| `cs` | Input | Active-low chip select |
| `mosi` | Input | Serial data from master |
| `din` | Input | Parallel word transmitted from slave to master |
| `miso` | Output | Serial data to master |
| `dout` | Output | Parallel word received from master |
| `done` | Output | Transfer-complete pulse |

### Slave Edge Selection

The slave computes which physical `sclk` edge is the sample edge:

```systemverilog
localparam bit SAMPLE_ON_POSEDGE = (CPOL == CPHA);
```

This maps the four SPI modes to the correct sampling edge:

| CPOL | CPHA | Sample Edge |
|---:|---:|---|
| 0 | 0 | Posedge |
| 0 | 1 | Negedge |
| 1 | 0 | Negedge |
| 1 | 1 | Posedge |

### Synthesizable Single-Edge Design

A tempting SPI slave implementation is to use a dual-edge block like:

```systemverilog
always_ff @(posedge sclk or negedge sclk)
```

That may simulate, but it is generally not synthesizable for FPGA flip-flops. This design avoids that issue by using a `generate` block. At elaboration time, it chooses one of two implementations:

- Sample on `posedge sclk`, shift on `negedge sclk`
- Sample on `negedge sclk`, shift on `posedge sclk`

Each branch uses ordinary single-edge `always_ff` blocks, making the slave more synthesis-friendly.

### Chip Select Behavior

`cs` is active low:

- When `cs` is high, the slave is deselected and internal counters are reset.
- When `cs` falls, the slave latches `din` into `tx_shift`.
- For `CPHA = 0`, the first `MISO` bit is preloaded immediately on `negedge cs` because the first sample edge occurs before any shift edge.

### Slave Receive Behavior

On each sampling edge, the slave shifts in `mosi`:

```systemverilog
rx_shift <= {rx_shift[DW-2:0], mosi};
```

When `DW` bits have been received, the slave updates:

```systemverilog
dout <= {rx_shift[DW-2:0], mosi};
done <= 1'b1;
```

## 7. Top Module

### Module Name

```systemverilog
top
```

The top-level module instantiates one master and one slave and connects:

```text
master.sclk -> slave.sclk
master.cs   -> slave.cs
master.mosi -> slave.mosi
slave.miso  -> master.miso
```

### Top-Level Ports

| Port | Direction | Description |
|---|---|---|
| `clk` | Input | System clock |
| `rst` | Input | Reset |
| `newd` | Input | Start transfer |
| `m_din` | Input | Master transmit word |
| `s_din` | Input | Slave transmit word |
| `m_dout` | Output | Word received by master |
| `s_dout` | Output | Word received by slave |
| `master_done` | Output | Master transfer done |
| `slave_done` | Output | Slave transfer done |

Expected full-duplex result:

```text
m_din -> s_dout
s_din -> m_dout
```

## 8. Source File: `tb.sv`

The testbench is class-based and self-checking. It runs the same verification environment across all four CPOL/CPHA combinations.

### Testbench Structure

```text
tb_top
+-- tb_spi #(CPOL=0, CPHA=0)
+-- tb_spi #(CPOL=0, CPHA=1)
+-- tb_spi #(CPOL=1, CPHA=0)
+-- tb_spi #(CPOL=1, CPHA=1)
```

Each `tb_spi` instance contains:

- SPI interface
- DUT instance
- Environment object

The tests are started one at a time from `tb_top` so console output remains readable.

## 9. Verification Components

### `spi_if`

The interface groups all DUT-facing signals:

- `clk`
- `rst`
- `newd`
- `m_din`
- `s_din`
- `m_dout`
- `s_dout`
- `done`
- debug taps for `sclk`, `cs`, `mosi`, and `miso`

It also includes a simple protocol check to ensure `done` does not remain high for more than one `clk` cycle.

### `transaction`

The transaction stores:

- Random `m_din`
- Random `s_din`
- Captured `m_dout`
- Captured `s_dout`

This represents one full-duplex SPI transfer.

### `generator`

The generator creates randomized transactions and sends them to the driver through a mailbox. It waits for the scoreboard to finish before generating the next transaction.

### `driver`

The driver:

1. Applies reset.
2. Drives `m_din` and `s_din`.
3. Pulses `newd`.
4. Sends expected values to the scoreboard.
5. Waits for `done`.

### `monitor`

The monitor waits for the transfer-complete pulse, samples `m_dout` and `s_dout`, and forwards the observed transaction to the scoreboard.

A small `#1` delay is used after `done` to avoid same-delta scheduling races between the master and slave outputs in simulation.

### `scoreboard`

The scoreboard checks both full-duplex paths:

```text
expected m_din == actual s_dout
expected s_din == actual m_dout
```

It counts matched transfers and mismatches for each CPOL/CPHA mode.

### `environment`

The environment connects all verification components together using mailboxes and events, then runs:

- reset
- generation
- driving
- monitoring
- checking
- reporting

## 10. Test Plan

The current test plan is:

| Test | Description |
|---|---|
| Mode 0 | CPOL = 0, CPHA = 0 |
| Mode 1 | CPOL = 0, CPHA = 1 |
| Mode 2 | CPOL = 1, CPHA = 0 |
| Mode 3 | CPOL = 1, CPHA = 1 |
| Random data | 5 randomized full-duplex transfers per mode |
| Scoreboard check | Verifies master-to-slave and slave-to-master paths |
| Done pulse check | Ensures `done` does not stay high for multiple clock cycles |

Total default coverage:

```text
4 modes * 5 transactions = 20 full-duplex transactions
```

## 13. How To Run

Use a simulator that supports SystemVerilog classes and interfaces.

Generic QuestaSim-style flow:

```sh
vlog design.sv tb.sv
vsim tb_top
run -all
```

Vivado xsim flow:

1. Create or open a Vivado project.
2. Add `design.sv` as a design source.
3. Add `tb.sv` as a simulation source.
4. Set `tb_top` as the simulation top.
5. Run behavioral simulation.

The testbench also creates:

```text
dump.vcd
```

which can be opened in GTKWave or another waveform viewer if the simulator supports VCD dumping.

## 14. Important Implementation Notes

- The design transfers data MSB first.
- `cs` is active low.
- `newd` should be pulsed to start a transfer.
- `CLK_DIV` controls the SCLK half-period, not the full period.
- The master uses the system clock; the slave operates from `sclk`, as a real SPI slave would.
- The slave design avoids non-synthesizable dual-edge sequential logic by selecting a single-edge implementation at elaboration time.
- The current top-level is built for one master and one slave. Multi-slave SPI would require separate chip-select signals for each slave.
